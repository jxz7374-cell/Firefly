# 部署方案

## 1. 项目部署目标

本系统需要同时满足两个目标：

- 满足课程作业要求，核心业务逻辑由 `C` 语言实现
- 能通过网页访问，支持展示增删改查与统计功能

因此，本项目采用如下部署结构：

```text
浏览器 <-> Node.js Web 服务 <-> C 语言核心程序 <-> students.txt
```

如果需要“长期免费、每天都能打开”的公网访问，更推荐静态部署版本：

```text
浏览器 <-> GitHub Pages 静态页面 <-> 浏览器本地存储 localStorage
```

## 2. 当前为什么通常只有自己能访问

如果你只是在自己电脑上运行：

```powershell
node server.js
```

默认情况下一般只有以下几种访问方式：

- 你自己通过 `http://localhost:3000` 访问
- 同一局域网用户通过 `http://你的局域网IP:3000` 访问

如果别人访问不到，常见原因是：

- 没有使用你的局域网 IP
- Windows 防火墙阻止了外部访问 3000 端口
- 电脑和访问者不在同一个局域网
- 家用路由器没有做公网端口映射

## 3. 三种可用部署方式

### 方案 A：局域网演示版

适合：

- 课堂展示
- 宿舍/实验室同一 Wi-Fi 下演示

步骤：

1. 在你的电脑上启动项目
2. 查询本机 IPv4 地址：`ipconfig`
3. 让同一网络下的同学访问 `http://你的IPv4地址:3000`
4. 如果访问失败，放行 Windows 防火墙 3000 端口

优点：

- 最简单
- 不需要购买服务器

缺点：

- 只能同一局域网访问
- 你的电脑必须一直开机

### 方案 B：云服务器公网部署版

适合：

- 所有人通过互联网访问
- 答辩展示时直接发链接

基本思路：

1. 购买一台云服务器
2. 安装 `Node.js`、`gcc`、`Nginx`
3. 上传项目文件
4. 执行 `node server.js`
5. 用 Nginx 反向代理到 `3000` 端口
6. 开放云服务器安全组中的 `80`、`443` 端口
7. 绑定域名后即可公开访问

优点：

- 所有人都能访问
- 更稳定
- 适合课程答辩、作品展示

缺点：

- 需要服务器费用
- 需要基本 Linux 部署操作

### 方案 D：GitHub Pages 免费静态版

适合：

- 没有预算
- 希望网址长期可访问
- 主要用于作业展示和演示

实现方式：

- 将网页改造成纯前端静态页面
- 学生成绩数据保存在浏览器本地 `localStorage`
- 部署到 GitHub Pages

优点：

- 免费
- 稳定
- 不依赖后端服务长期运行

缺点：

- 数据不会在不同用户之间同步
- 不再由服务器实时调用 C 程序

### 方案 C：内网穿透临时展示版

适合：

- 短时间给老师或同学演示
- 不想买服务器

基本思路：

- 在自己电脑运行项目
- 使用内网穿透工具把本地 `3000` 端口映射成公网地址

优点：

- 上手快
- 不需要长期维护服务器

缺点：

- 稳定性一般
- 免费线路可能慢
- 不适合长期公开使用

## 4. 推荐部署方案

如果你只是交作业，我建议：

- 课堂演示：使用方案 A
- 需要给老师一个外网链接：使用方案 B

因为本项目依赖 `Node.js` 和本地 `gcc` 编译的 C 程序，所以相比纯静态网页，更适合部署到云服务器，而不是静态托管平台。

## 5. Windows 本机局域网开放方法

### 第一步：启动项目

```powershell
cd C:\Users\Thinkbook\Desktop\student
node server.js
```

启动成功后，终端会显示：

- `http://localhost:3000`
- `LAN access: http://你的局域网IP:3000`

### 第二步：查看本机 IP

```powershell
ipconfig
```

记录 `IPv4 Address`，例如：

```text
192.168.1.20
```

则别人访问地址为：

```text
http://192.168.1.20:3000
```

### 第三步：放行防火墙端口

可在管理员 PowerShell 中执行：

```powershell
New-NetFirewallRule -DisplayName "StudentSystem3000" -Direction Inbound -Protocol TCP -LocalPort 3000 -Action Allow
```

## 6. 云服务器部署步骤

以下以 Linux 服务器为例：

### 6.1 安装运行环境

```bash
sudo apt update
sudo apt install -y nodejs npm gcc nginx
```

### 6.2 上传项目

将整个项目目录上传到服务器，例如：

```text
/var/www/student
```

### 6.3 启动服务

```bash
cd /var/www/student
node server.js
```

建议后续使用 `pm2` 或 `systemd` 常驻运行。

### 6.4 配置 Nginx

参考 [deploy/nginx.student-system.conf](C:/Users/Thinkbook/Desktop/student/deploy/nginx.student-system.conf)

核心思想是：

- 外部访问 `80/443`
- Nginx 转发到本地 `3000`

### 6.5 开放端口

需要在云平台安全组放行：

- `80`
- `443`
- 如需远程登录还需 `22`

## 7. Railway 公网部署步骤

这是目前最省事的一种方式，适合你这个 `Node.js + C + Dockerfile` 项目。

根据 Railway 官方文档：

- Railway 支持从 GitHub 部署项目  
  https://docs.railway.com/quick-start
- Railway 检测到 `Dockerfile` 时，会按 `Dockerfile` 构建并部署  
  https://docs.railway.com/deploy/dockerfiles
- Railway 会给服务注入 `PORT` 环境变量，你的程序已经兼容  
  https://docs.railway.com/reference/healthchecks
- 公开访问可在 `Settings -> Networking -> Public Networking` 中生成域名  
  https://docs.railway.com/reference/public-networking

### 部署步骤

1. 把当前项目上传到 GitHub
2. 注册并登录 Railway
3. 在 Railway 里点击 `New Project`
4. 选择 `Deploy from GitHub repo`
5. 选择你的这个仓库
6. Railway 会检测到仓库中的 `Dockerfile` 并自动构建
7. 等部署完成后，进入服务设置页
8. 在 `Networking -> Public Networking` 点击 `Generate Domain`
9. 得到一个 `*.railway.app` 的公网地址
10. 打开这个地址即可让别人访问

### 建议的附加设置

- Health Check Path：`/health`
- Root Directory：项目根目录
- Start Command：通常不需要单独填，因为 `Dockerfile` 里已经有 `CMD ["node", "server.js"]`

### 如果部署失败，优先检查

- GitHub 仓库是否包含 `Dockerfile`
- `students.txt` 是否已提交
- Railway 日志里是否提示编译 `gcc` 失败
- 服务是否监听 `PORT` 环境变量

## 8. GitHub Pages 免费部署步骤

这是目前最适合学生长期免费展示的方案。

### 项目现状

当前项目已经提供了静态部署入口：

- 根目录跳转页：`index.html`
- 实际页面：`public/index.html`

部署前建议：

1. 把修改后的文件推送到 GitHub

### GitHub Pages 配置步骤

1. 打开你的 GitHub 仓库
2. 进入 `Settings`
3. 点击左侧 `Pages`
4. 在 `Build and deployment` 中选择：
   - Source: `Deploy from a branch`
   - Branch: `main`
   - Folder: `/ (root)`
5. 点击 `Save`
6. 等待 GitHub 构建完成
7. 打开生成的网址

生成的网址通常类似：

```text
https://你的GitHub用户名.github.io/student-score-system/
```

### 为什么这个方案适合你

- 不需要服务器费用
- 不受 Railway 试用额度限制
- 网页可以长期在线
- 非常适合作业演示和答辩展示

## 9. 是否可以做到“所有人都能访问”

可以，但前提是满足以下任意一种：

- 你的项目部署在有公网 IP 的云服务器上
- 你的家用路由器做了公网端口映射，并且拥有可访问的公网地址
- 你使用了可用的内网穿透服务

单纯在自己电脑上运行，并不等于全网公开。
