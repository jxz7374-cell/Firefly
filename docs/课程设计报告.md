# 学生成绩管理系统课程设计报告

## 1. 课题名称

学生成绩管理系统的设计与实现

## 2. 课题背景与意义

在教学管理中，学生成绩信息的录入、修改、查询和统计是最基础也是最常见的工作内容。如果采用手工方式记录成绩，不仅效率较低，而且容易出现重复录入、统计错误和信息丢失等问题。为了解决这些问题，设计并实现一个学生成绩管理系统具有较强的实际意义。

本课题以学生成绩管理为核心，实现学生信息的增加、删除、查询、修改及成绩统计等功能。同时，为了满足课程设计对 C 语言能力的考察要求，系统核心逻辑采用 C 语言实现；为了提升系统展示效果和交互体验，系统增加了 Web 页面作为操作界面，使系统更直观、更易演示。

## 3. 需求分析

### 3.1 基础功能需求

- 添加学生信息：包括学号、姓名、高数、英语、C 语言成绩
- 删除指定学号的学生记录
- 按学号或姓名查询学生信息
- 修改学生基本信息或某门课程成绩
- 启动时从 `students.txt` 加载数据
- 退出或更新后将数据保存到 `students.txt`

### 3.2 统计功能需求

- 统计各科平均分、最高分、最低分
- 计算每位学生的总分和平均 GPA
- 按成绩区间统计人数分布
- 生成班级成绩排名表
- 统计不及格学生名单及不及格科目
- 计算各科及格率与优秀率

## 4. 总体设计

### 4.1 系统架构

系统采用三层结构：

1. 表现层：使用 HTML、CSS、JavaScript 构建网页界面
2. 服务层：使用 Node.js 接收浏览器请求并调用 C 程序
3. 核心层：使用 C 语言实现学生管理和成绩统计逻辑

系统结构如下：

```text
浏览器
  ↓
Node.js Web 服务
  ↓
C 语言核心程序
  ↓
students.txt
```

### 4.2 数据结构设计

系统中定义学生结构体，包含以下字段：

- 学号 `id`
- 姓名 `name`
- 高数成绩 `math`
- 英语成绩 `english`
- C 语言成绩 `c_language`

每条记录保存在文本文件中，格式如下：

```text
学号|姓名|高数|英语|C语言
```

例如：

```text
2023001|Alice|92.00|88.00|95.00
```

## 5. 详细设计

### 5.1 添加功能

用户在网页中输入学生信息后，Node.js 将数据传递给 C 程序。C 程序先检查学号是否重复，再验证成绩范围是否合法，最后把新数据写入内存并保存到 `students.txt`。

### 5.2 删除功能

根据输入的学号查找对应学生，如果存在则删除该记录，并将后续数组元素前移，最后重新保存文件。

### 5.3 查询功能

- 按学号查询：精确匹配
- 按姓名查询：模糊匹配

查询结果返回网页并展示在表格中。

### 5.4 修改功能

系统支持修改：

- 学号
- 姓名
- 高数成绩
- 英语成绩
- C 语言成绩

修改完成后立即保存到文件。

### 5.5 统计功能

系统统计模块包括：

- 各科平均分、最高分、最低分
- 及格率与优秀率
- 学生总分与 GPA
- 按总分生成排名表
- 成绩区间分布统计
- 不及格学生及科目列表

## 6. 关键算法说明

### 6.1 总分计算

每位学生总分等于三门课程成绩之和：

```text
总分 = 高数 + 英语 + C语言
```

### 6.2 GPA 计算

本系统采用分段绩点法，将每门成绩映射为绩点，再求平均值得到 GPA。

### 6.3 排名算法

系统按照总分从高到低排序；若总分相同，则优先比较高数成绩；若仍相同，再比较学号。

### 6.4 不及格判断

若某门课程成绩低于 60 分，则视为该科不及格，并记录到不及格名单中。

## 7. 系统实现环境

- 操作系统：Windows / Linux
- 编程语言：C、JavaScript
- 前端技术：HTML、CSS、JavaScript
- 服务端：Node.js
- 编译器：GCC
- 数据存储：文本文件 `students.txt`

## 8. 系统测试

### 8.1 测试内容

- 添加学生信息是否成功
- 删除指定学号是否成功
- 查询功能是否正确返回结果
- 修改成绩后是否同步更新
- 重新启动后数据是否仍然存在
- 各类统计结果是否正确

### 8.2 测试结果

经测试，系统能够正确完成学生信息管理及成绩统计功能，网页操作与 C 语言核心逻辑联通正常，数据可正确保存和加载，系统运行稳定，满足课程设计要求。

## 9. 系统特点

- 结构清晰，网页与 C 核心分工明确
- 既满足课程作业要求，又便于演示
- 支持数据持久化
- 支持多种统计分析功能
- 界面直观，操作方便

## 10. 不足与改进方向

虽然本系统已经实现了主要功能，但仍存在一些不足：

- 当前数据存储采用文本文件，不适合超大规模数据
- 用户权限控制尚未实现
- 目前未加入登录认证功能
- 图表展示可进一步丰富

后续可以考虑：

- 接入数据库
- 增加登录模块
- 增加导出 Excel 或 PDF 功能
- 增加图表化统计页面

## 11. 总结

本课题完成了一个基于 C 语言核心与 Web 页面相结合的学生成绩管理系统。系统实现了学生信息的增删改查以及成绩统计、排名、不及格分析等功能，并通过 `students.txt` 实现数据持久化。该系统不仅锻炼了 C 语言程序设计能力，也体现了前后端结合的实际应用能力，具有一定的实用价值和展示意义。
